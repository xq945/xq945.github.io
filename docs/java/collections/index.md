# Java集合


## Q：为什么 HashMap 选择红黑树而不是 AVL 树？

HashMap 在 JDK 8 中，当链表长度达到 8 且数组容量达到 64 时，会将链表转换为红黑树，而不是 AVL 树，主要原因如下：

1. **红黑树属于弱平衡结构**
    - 不要求左右子树高度差始终 ≤ 1。
    - 只需满足红黑树的颜色规则即可保持 O(log n) 的高度。

2. **插入、删除效率更高**
    - AVL 为了维持严格平衡，需要更频繁地进行旋转。
    - 红黑树大多数情况下只需变色，旋转次数更少。

3. **综合性能更好**
    - AVL 查询效率略高。
    - 红黑树查询效率同样为 O(log n)，但插入、删除代价更低，更适合 HashMap 这种频繁 put/remove 的场景。

HashMap 选择红黑树而不是 AVL 树，是因为红黑树属于弱平衡二叉搜索树，虽然查询效率略低于 AVL，但插入和删除时需要的旋转次数更少，综合性能更优。HashMap 的桶中不仅有查询，还有大量的 put 和 remove 操作，因此红黑树更适合作为底层数据结构。
